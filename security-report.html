üî¥ CRITICAL Vulnerabilities

SEC-001: Massive Database Data Exposure

Severity: CRITICAL | CVSS: 7.5 | Status: Open

API endpoints expose the complete internal database structure, including commercially sensitive fields that should never be public.

Exposed Sensitive Data:

  • ProductCost - Internal cost (profit margin)
  • StockQuantity - Real-time exact inventory
  • WarehouseId - Logistics infrastructure
  • VendorId - Internal suppliers
  • AdminComment - Administrative notes
  • TaxCategoryId - Tax classification

Proof of Concept:

curl "https://lcom-nextjsapp.happymeadow-56d1104c.centralus.azurecontainerapps.io/api/products/1966" # Returns 2,100+ lines of JSON with complete product data

Impact: Industrial espionage, inventory intelligence, system reconnaissance

SEC-002: IDOR - Insecure Direct Object Reference

Severity: CRITICAL | CVSS: 8.1 | Status: Open

The API allows access to any product by simply incrementing the numeric ID, without permission validation or authentication.

# Sequential access without restrictions for id in 1 2 3 100 1000 2000 9999; do curl "https://.../api/products/$id" done # All return HTTP 200 with complete data

Remediation: Implement rate limiting, require authentication, use UUIDs instead of sequential IDs

üü† HIGH Severity Vulnerabilities

SEC-003: Cross-Site Scripting (XSS) Reflected

Severity: HIGH | CVSS: 6.1

SEC-004: No Rate Limiting

Severity: HIGH | CVSS: 5.3

The API does not implement rate limiting, allowing unlimited requests from the same IP.

SEC-005: Missing Security Headers

Severity: HIGH | CVSS: 5.0

Missing: X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security, Content-Security-Policy
